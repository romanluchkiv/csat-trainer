# CSAT Trainer — v9.1 (patch on top of v9)

## What's NEW in v9.1

Only fixes Roman explicitly requested. Nothing else changed.

### UI fixes (trainer.py)

1. **12Go logo** — real logo image (12go_logo.jpg, 300x300) replaces the green text badge in the header.

2. **Case context block removed** from the right sidebar. The two duplicated/distracting blocks are gone:
   - Top: `#### Case context` + case title + alleging paragraph
   - Bottom: italicized `additional_notes` paragraph

3. **Base font size increased** throughout the app for readability — body text now 16-17px (was ~13-14px), headers proportionally larger.

4. **Live Coach feedback REMOVED** — the auto-expanded "💡 Coach feedback on your last reply" block that appeared after every public reply is gone. Also removes the per-turn `get_coach_feedback` API call (saves tokens). Full per-turn breakdown is still available at session end via the **Show detailed manager view** toggle, generated by the structured Judge tool.

5. **Yellow internal note text now readable** — added `color: #1a1a1a` (dark text) to the three internal note divs (agent_internal, ops, it). Previously white text on yellow was invisible on Streamlit dark theme.

### Prompt fixes (prompts.py)

6. **CLIENT_SYSTEM_PROMPT — fact consistency rule.** Customer LLM must stay consistent with concrete facts from its own opening message (route, locations, where they were physically). May add emotional color but NOT contradicting facts. Fixes the "Hurghada → Sharm in opening, then 'I was IN CAIRO'" hallucination.

7. **CLIENT_SYSTEM_PROMPT — attachment marker rule.** When customer "sends" evidence in any follow-up reply (photos, receipts, screenshots), the message must end with `[Attachments: description]` so trainer.py renders the placeholder box. Already worked in opening messages, now also in follow-ups.

### Case data fix (enrich_v9_metadata.py)

8. **Case 2 (Egypt safety) max_turns: 8 → 7**, matching the original plan (7 for complex cases). Applied automatically by re-running the enrichment script.

---

## What is NOT changed

Everything else in v9 remains exactly as it was: `[END_CONVERSATION]` logic, IT/Ops routing by greeting, structured Judge (tool_use), emotional acknowledgment block, agent name dropdown, BID + pickup/dropoff in Booking Info, manager view toggle, etc.

## Installation

### Step 1 — Stop Streamlit
Ctrl+C in the terminal where it's running.

### Step 2 — Replace 3 files + add logo
Inside `~/Desktop/csat_trainer/`:
- Replace `trainer.py` with the v9.1 version
- Replace `prompts.py` with the v9.1 version
- Replace `enrich_v9_metadata.py` with the v9.1 version
- Add `12go_logo.jpg` (new file — drop it directly into the folder)

`agent_roster.csv` stays as you have it.

### Step 3 — Re-run enrichment to apply Case 2 max_turns fix
```bash
cd ~/Desktop/csat_trainer
source venv/bin/activate
python enrich_v9_metadata.py
```

You should see:
```
--- case_636721.json ---
  Applied fixes: max_turns=7
  All v9 fields already present (use --force to overwrite)
  Saved.
```

The script is safe — it only updates what's missing or what's in PER_CASE_FIXES. Your enriched customer names, BIDs, pickup/dropoff stay as they are.

### Step 4 — Launch
```bash
streamlit run trainer.py
```

### Step 5 — Verify
- Real 12Go logo in the header (not the green text badge)
- No "Case context" block on the right
- Booking Info is more readable (larger font)
- No coach feedback expander appearing after each public reply
- Internal notes: yellow background with dark, readable text
- If you let customer "send" evidence in follow-up, you should see the attachment placeholder box (not just prose)
- Case 2 should show "Public reply turn 1 of 7"
